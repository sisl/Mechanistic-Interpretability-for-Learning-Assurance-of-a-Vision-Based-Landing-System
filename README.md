# Mechanistic Interpretability for a Vision-Based Landing System

Reproducible code and figures for the DASC 2026 paper. The full pipeline (one
target per paper section) is driven by the `Makefile`; `src/` is the importable
library (data, model, losses, evaluation) and `scripts/` are the typer CLIs that
compose it. K-SVD / sparse-coding steps run in Julia (`scripts/julia/`, with the
vendored `libs/KSVD.jl`).

## Setup

```bash
uv sync                              # Python env (pinned by uv.lock)
julia --project=. -e 'using Pkg; Pkg.instantiate()'   # Julia deps (Manifest.toml)
```

## Run

```bash
DRY=1 make all          # CPU smoke run end-to-end (~1 min, synthetic-scale data)
make all VARIANT=pretrained SEED=0 DEVICE=cuda:0    # one real (variant, seed)
SEEDS="0 1 2 3 4" ./scripts/run_sweep.sh            # full multi-seed sweep
make aggregate          # collapse per-seed results into figs/aggregated.json
```

`make help` lists every target and its paper-section mapping. Experiment outputs
(`data/`, `models/`, `results/`, `logs/`) are gitignored and regenerated by the
pipeline; the figure assets the paper consumes live under `figs/`.
